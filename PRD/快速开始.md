# OpenSpec 快速开始指南

> 5 分钟快速上手 OpenSpec 需求管理

## 🚀 最简单的提需求方式

在 Cursor 中，直接输入：

```
/openspec-proposal [你的需求]
```

**示例：**
```
/openspec-proposal 我想添加一个新的订单类型：冰山订单
```

AI 会自动帮你：
- ✅ 查看现有规范
- ✅ 创建提案文档
- ✅ 生成任务清单
- ✅ 验证格式

## 📋 完整流程（3 步）

### 1️⃣ 提需求 → 创建提案

```
/openspec-proposal 你的需求描述
```

**结果**：在 `openspec/changes/` 下创建提案目录

### 2️⃣ 评审通过 → 实现

```
/openspec-apply <change-id>
```

**结果**：按任务清单实现功能

### 3️⃣ 部署完成 → 归档

```
/openspec-archive <change-id>
```

**结果**：更新规范文档，归档变更

## 💡 常用命令

```bash
# 查看所有活动变更
openspec list

# 查看所有规范
openspec list --specs

# 查看具体变更
openspec show <change-id>

# 验证变更格式
openspec validate <change-id> --strict
```

## ❓ 什么时候需要提案？

**需要提案**：
- ✅ 添加新功能
- ✅ 修改 API/数据结构
- ✅ 架构变更

**不需要提案**：
- ❌ Bug 修复
- ❌ 拼写错误
- ❌ 配置变更

---

**提示**：不确定是否需要提案？建议创建，更安全！
